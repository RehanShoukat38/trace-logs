# Implementation Plan - Dynamic Pricing Agent

Create a new agent to calculate dynamic pricing for service requests using multiple factors including complexity, urgency, distance, and demand.

## User Review Required

> [!NOTE]
> The agent will strictly follow the provided 8-factor pricing formula and fairness checks. Alternate options will be generated by simulating scenarios like reduced urgency.

## Proposed Changes

### Dynamic Pricing Agent

#### [NEW] [dynamicPricingAgent.ts](file:///c:/Users/satti/Desktop/Hackathon%20Project/guardian-ai/backend/functions/src/agents/dynamicPricingAgent.ts)
- Define `PricingInput` and `PricingOutput` interfaces.
- Implement `DynamicPricingAgent` class.
- Implement `calculatePrice` method:
    - Base Price: `hourly_rate * estimated_hours`.
    - Complexity Adjustment: 0%, 20%, or 50% based on level.
    - Distance Cost: PKR 10 per km beyond 3 km.
    - Urgency Surge: 0% to 30% based on level.
    - Demand Pricing: 0% to 25% based on current demand.
    - Time Slot Pricing: Night (15%), Weekend (10%), Holiday (20%).
    - Loyalty Discount: 0% to 15% based on tier.
    - Visit Fee: Flat PKR 200.
    - Fairness Checks:
        - Cap maximum surge at 50% of base.
        - Ensure provider earning is at least 70% of total.
    - Generate `alternate_options` (e.g., scheduling for later to remove urgency surge).
